# ADR 0005: 이중 지출 차단 (Double-Count Prevention) 정책

## 상태
초안 (Proposed)

## 컨텍스트
현재 시스템에서는 개인 지출(Personal)과 사업 지출(Business)이 명확히 분리되지 않아, 사업비로 분류된 금액이 개인 가계부 통계에도 합산되는 문제가 발생함. 이로 인해 실제 순자산(Net Worth)이 이중으로 차감되어 나타나며, 정확한 현금 흐름 파악이 어려움.

## 결정
사업비로 태깅된(`allocation_status`가 사업 관련인) 거래 내역에 대해 `excluded_from_personal` 플래그를 도입하고, 모든 개인용 통계 API에서 이를 필터링하도록 결정함.

1.  **데이터 무결성**: 하나의 거래(Transaction)는 개인 또는 사업 중 하나의 레이어에만 최종적으로 귀속됨을 원칙으로 함.
2.  **명시적 플래그**: `allocation_status` 필드에만 의존하지 않고, 명시적인 `excluded_from_personal` 불리언 필드를 추가하여 쿼리 성능과 가독성을 확보함.
3.  **UI 피드백**: 가계부 뷰에서 해당 항목을 삭제하는 대신 '제외됨' 상태를 시각적으로 유지하여 사용자가 데이터의 흐름을 추적할 수 있게 함.

## 결과
- **긍정적**: 순자산 통계의 정확성 확보, 개인/사업 지출의 명확한 회계적 분리.
- **부정적**: 통계 조회 쿼리에 필터 조건이 추가되어 복잡도가 약간 증가함.
- **후속 작업**: 기존 사업비 데이터에 대한 마이그레이션 스크립트 실행 필수.
