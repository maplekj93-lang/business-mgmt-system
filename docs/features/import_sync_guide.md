# 가계부 엑셀 임포트 동기화 가이드 기능 명세 (Import Sync Guide)

> **작성일:** 2026-03-05
> **목표:** 사용자가 각 결제수단(통장/카드/에셋)별 엑셀 내역을 임포트할 때 중복 입력 방지 및 편의 제공

---

## 1. 기능 개요
여러 결제수단을 각각 기간별로 엑셀로 내려받다 보면, "어디까지 임포트 했더라?" 헷갈려 중복되거나 누락된 구간이 생길 확률이 높습니다. 
이를 방지하고자 각 자산별 **"마지막 거래일자"**와 **"마지막 실제 정보 업데이트(동기화) 일자"**를 모두 확인하여, **"어느 날짜부터 엑셀을 다운로드 받아야 하는지"** 명시적으로 가이드해주는 기능입니다.

## 2. 주요 제공 기능 (Features)

### 2.1. 최근 거래일 및 마지막 업데이트(동기화) 날짜 표시
- **마지막 거래일자 (`Last Transaction Date`):** 해당 자산에 실제로 돈이 결제/입금된 최신 날짜.
- **마지막 업데이트 일자 (`Last Synced Date`):** 사용자가 해당 자산의 엑셀 데이터를 업로드하여 시스템에 업데이트를 마친 실제 시스템 날짜. (거래가 드문 '청약통장' 등의 경우, 마지막 거래일은 1년 전이더라도 어제 업데이트했다면 안심할 수 있음)

### 2.2. 임포트 추천 시작일 안내 (Suggested Import Start Date)
- 두 날짜(마지막 거래일, 마지막 업데이트 일자)를 종합하여, 안전하게 다운로드를 시작할 날짜를 안내.
- 예: "👉 **권장 시작일: `2026. 02. 26`일부터 받아주세요.** (마지막 파일 업로드: 오늘 오상 9시)"

### 2.3. 카드/통장사 바로가기 빠른 링크
- 각 은행사별 엑셀 내역 추출 페이지(PC 웹뱅킹 등) 접속을 원활하게 돕기 위한 퀵 링크 (예: 농협 바로가기, 신한 파트너스 등).

---

## 3. 핵심 고려사항 (Edge Cases & Considerations)

### 3.1. 겹치는 구간 중복 데이터 방어 (Idempotency)
- 엑셀 다운로드 시 사용자가 안전하게 며칠 전부터 여유 있게 겹쳐서 다운받아 올릴 확률이 높습니다.
- **해결 방안:** 단순히 날짜로 거르지 않고, **고유 해시(날짜+시간+금액+적요+자산ID)** 조합 등을 만들어 이미 들어있는 내역은 에러를 내지 않고 조용히 무시(Upsert 또는 Ignore)하는 방어 로직이 반드시 필요합니다.

### 3.2. 해외 결제 및 승인/매입 지연 (Pending Transactions)
- 신용카드의 경우 오늘 결제해도 며칠 뒤에 '매입' 상태가 되어 금액(환율)이나 적요가 바뀌는 경우가 있습니다.
- **해결 방안:** 가이드 화면에 "오늘 결제건은 아직 확정되지 않았을 수 있으니 가급적 어제 내역까지만 다운받기를 권장합니다."라는 경고성 문구를 넣으면 좋습니다.

### 3.3. 수동 입력 내역과의 충돌 구분
- 같은 날 같은 통장에서 수동으로 입력해둔 내역이 있는데, 엑셀로 한 번 더 임포트될 경우 어설프게 겹칠 수 있습니다.
- **해결 방안:** 엑셀로 올린 건지, 손으로 쓴 건지 구분할 수 있는 `source` 컬럼(예: `EXCEL`, `MANUAL`) 정도를 두어, 나중에 수정하거나 롤백할 때 참고할 수 있도록 설계하면 좋습니다.

### 3.4. 시간대 (Timezone) 문제
- 은행마다 엑셀 날짜와 시간 형식이 천차만별입니다. 어떤 은행은 시/분/초가 없고 어떤 곳은 있습니다. 
- **해결 방안:** 00시 00분으로 일괄 통일할지, 아니면 가능한 한 원래 엑셀 시간을 한국표준시(KST)로 정확히 저장할지에 대한 포맷 파싱 규칙 정리가 필요합니다.

## 4. 데이터베이스 변경 필요 사항 (제안)

이 기능을 위해 차후 DB 스키마 쪽에 아래와 같은 컬럼이나 테이블 변경이 필요할 수 있습니다.

1. **`assets` 테이블:**
   - `last_synced_at` (TIMESTAMPTZ): 해당 자산의 엑셀 업로드를 마지막으로 완료한 실제 시각. 임포트 성공 시 갱신.

2. **`transactions` 테이블:**
   - `import_hash` (TEXT UNIQUE): 똑같은 데이터가 중복으로 들어오는 것을 방지하기 위한 해시 키.
   - `source` (TEXT): 데이터 출처. (기본값: 'MANUAL', 엑셀 시 'EXCEL_IMPORT')
