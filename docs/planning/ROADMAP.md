## Phase 0: 기반 구축 (DB Seed)

> **목표:** 코드를 짜기 전에 DB 데이터가 올바르게 세팅되어 있어야 한다.

### 0-1. 카테고리 시드 확인 및 정비
- `mdt_categories` 테이블에 MDT_CATALOG의 **대분류 → 소분류 계층**이 들어있는지 확인
- 없으면 Supabase MCP 통해 직접 시드 SQL 실행
- 이모지 포함 정확한 이름 필요 (파서가 이름으로 매칭하기 때문)

```
지출 대분류 (18개): 🏠 주거비, 🛍️ 물품구입비, 🍽️ 식비, ☕️ 커피, 🚌 교통비,
  💰 보험료, 🩺 건강, 🎬 문화생활, 📱 구독/서비스, 📞 통신비,
  👩💻 사업비, 💛 기부금, 🏛️ 세금, 🎁 경조/선물, 💰 저축,
  💇♀️ 꾸밈비, ✈️ 여행, ⚠️ 기타비용

수입 대분류 (2개): 💼 사업소득, 💵 금융소득
이체 대분류 (1개): 🔄 이체
```

### 0-2. 자산/계좌 시드
- `assets` 테이블에 MDT_CATALOG 섹션3 기준 계좌/카드 목록 입력
- 중요: 파서가 `출금계좌` 컬럼값으로 계좌를 매칭하기 때문에 이름이 정확해야 함

```
주요 계좌 (예시):
- 광준 / 사업자 계좌 / 기업은행  (asset_type: bank)
- 광준 / 사업자 삼성카드          (asset_type: credit_card)
- 쾅영 / 현대카드                  (asset_type: credit_card)
- 의영 개인 / 기업은행             (asset_type: bank)
- 광준 / 개인계좌 / 기업은행       (asset_type: bank)
...
```

### 0-3. 기본 분류 룰 시드 (mdt_allocation_rules)
- 자주 쓰는 가맹점 → 카테고리 매핑 룰 초기 세팅
- 예: `스타벅스` → `☕️ 커피 > 커피`, `쿠팡` → `🛍️ 물품구입비 > 기타물품`

**완료 기준:** Supabase 대시보드에서 카테고리 N개, 자산 N개 확인

### 1-1. 낼나 가계부 (가계부 기록) 임포트

### 1-2. 수입내역 임포트

## Phase 2: 은행/카드 명세서 멀티소스 임포트

> **목표:** 각 은행·카드사 명세서 파일을 드래그앤드롭으로 올리면 자동으로 처리된다.

### 2-1. 각 파서 검증 및 수정
docs 폴더의 실제 파일로 각 파서를 검증한다.

| 파서 | 파일 예시 | 핵심 확인 사항 |
|---|---|---|
| 삼성카드 | 삼성카드_사용내역.xlsx | 취소 건 net-zero 필터 |
| BC카드 | BC카드_이용내역.xlsx | 날짜 포맷, 승인번호 |
| 현대카드 | 현대카드_이용내역.xlsx | 할부 처리 |
| 카카오뱅크 | 카카오뱅크_입출금내역.csv | 이체 구분 |
| 기업은행 | 기업은행_거래내역.csv | EUC-KR 인코딩 |
| 아멕스 | 카드사용내역 시트 | 별도 프로파일 추가 필요 |

### 2-2. import 배치 관리 기능
- 임포트할 때 `import_batch_id` (UUID) 생성 → 저장
- 배치 단위 롤백 기능 추가 ("이 임포트 취소")
- 임포트 이력 화면 (`/settings/import` 개선)

### 2-3. 중복 임포트 방지
- 같은 거래를 두 번 넣을 때 막는 로직
- 기준: `(date + amount + description + account_hint)` 복합 키 해시
- 중복 시 "이미 있는 내역 N건 스킵됨" 안내

**완료 기준:** 은행/카드 명세서 업로드 → "N건 추가, M건 스킵(중복)" 결과 표시

## Phase 3: 자동 분류 시스템

> **목표:** 26년 이후 새 명세서를 올리면 과거 패턴 기반으로 자동 분류가 된다.

### 3-1. 미분류 Inbox UI 개선
현재 `/transactions/unclassified` 페이지를 실용적으로 개선.

- 가맹점 이름으로 그룹핑 (같은 곳 여러 건 한번에 처리)
- "이 가맹점은 항상 이 카테고리로" 원클릭 룰 등록
- 대분류 선택 → 소분류 자동 필터링 (2단계 셀렉트)
- 건너뛰기 / 나중에 기능

### 3-2. 키워드 룰 엔진 강화
- 분류 완료한 내역에서 자동 룰 생성 제안
  - "스타벅스를 ☕️ 커피로 20번 분류했습니다. 룰로 등록할까요?"
- `mdt_allocation_rules` 테이블에 저장
- 파서에서 규칙 적용 순서: 정확 매칭 → 부분 포함 → 수동

### 3-3. 카테고리 자동 제안 UI
- 미분류 거래의 description으로 유사한 과거 거래 찾기
- "비슷한 거래 N건이 [카테고리]로 분류되어 있어요" 표시
- 수락 / 다른 카테고리 선택

**완료 기준:** 새 명세서 업로드 후 분류율 70% 이상 자동화

## Phase 4: 비즈니스 ERP

> **목표:** 사업 관련 수입/지출을 가계부와 분리해서 프로젝트 단위로 관리한다.

### 4-1. 수입 내역 관리 (Income Pipeline)
**흐름:** 견적/계약 → 작업완료 → 입금확인 → 세금계산

화면: `/business/income`

| 상태 | 설명 |
|---|---|
| 진행중 | 촬영/작업 진행 중 |
| 작업완료 | 완료됐지만 미입금 |
| 입금전 | 청구서 발송 후 대기 |
| 정산완료 | 입금 확인 완료 |

- 계약금액 → 정산금액 (부가세 10% 또는 원천징수 3.3% 자동 계산)
- 업체별 미수금 현황 요약
- transactions 테이블의 income 거래와 연결 (입금 확인 시 자동 매칭)

### 4-2. 사업 지출 태깅
- 기존 가계부 거래 목록에서 사업용 항목 선택
- "이 지출은 사업비로 처리" → `allocation_status: 'business'` + `business_unit_id` 설정
- 사업 종류별 분류: 조명 크루 vs 디자인/포토
- 모드 전환(Personal → Business)시 사업 지출만 필터링

### 4-3. 프로젝트 관리
화면: `/business/projects`

- 프로젝트 단위 수익/비용 집계
  - 예: `W24 현대글로비스` — 수입 6,765,000 / 크루 인건비 X / 식대 Y / 순이익 Z
- 프로젝트에 거래 연결 (`project_id` 태깅)
- 프로젝트 상태: 진행중 / 완료 / 마감

### 4-4. 원천징수 (3.3%) 계산
- 일해주신분들 원천 시트 데이터 기반
- 지급 금액 입력 → 3.3% 자동 계산 → 실지급액 표시
- 지급 완료 처리 → 해당 지출 거래와 연결

### 4-5. 견적서 생성
화면: `/business/quote/new`

- 업체명, 작업 내용, 단가, 수량 입력
- 계약금액 / 부가세 / 최종금액 자동 계산
- PDF 출력 (A4 포맷, 사업자 정보 포함)
- 생성한 견적 → 수입 내역 파이프라인으로 자동 연결

## Phase 5: 대시보드 & 리포트 완성

> **목표:** 개인과 사업 재무 현황을 한눈에 파악할 수 있는 완성된 대시보드

### 5-1. Personal 트랙 대시보드
- 이번 달 수입 / 지출 / 순수입
- 카테고리별 지출 도넛 차트 (상위 5개 + 기타)
- 월별 추세 그래프 (최근 12개월)
- 예산 대비 실지출 (카테고리별 진행 바)
- 순자산 변화 (자산 합계 - 부채 합계)

### 5-2. Business 트랙 대시보드
- 미입금 수입 현황 (총 미수금액)
- 이번 달 사업 수입 / 사업 지출 / 순이익
- 프로젝트별 수익성 요약
- 업체별 거래 현황

### 5-3. Total 통합 뷰
- 개인 + 사업 합산 순자산
- 자산별 잔고 현황 (계좌/카드별)
- 연간 재무 흐름 타임라인

### 5-4. 월간 리포트
- 자동 생성: 매월 말 기준 집계
- 주요 지출 카테고리 Top 5
- 전월 대비 증감
- PDF 내보내기
