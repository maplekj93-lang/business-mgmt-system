# Business ERP (Phase 4) 설계 및 요구사항 명세서

> **작성일:** 2026-03-05
> **목표:** 쾅영부부 개인 가계부와 분리된 완전한 비즈니스(프리랜서) ERP 시스템 구축

---

## 0. 사업 주체 현황 정리

이 시스템은 아래 **3개의 사업 주체**를 관리합니다. 설계 전 명확히 구분해야 합니다.

| 주체 | 담당자 | 직종 | 수입 유형 |
|------|--------|------|---------|
| **광준 개인사업** | 광준 | 프리랜서 조명감독 + 영상스케치 촬영 | 세금계산서(사업자) 발행. 진행비(주차/주유 등) 포함 정산. 크루 인건비 총괄 가능. |
| **의영 개인사업** | 의영 | 프리랜서 그래픽/공간 디자이너 | 프리랜서 계약 |
| **공동 사진사업** | 공동 (광준+의영) | 사진 촬영/스튜디오 | 프로젝트 계약 |

### 수입 유형 정의

- **퍼스트 (일당)**: 광준이 현장에 하루 단위로 나가는 형태. 드라마/CF/광고 촬영 조명 스태프.
  - **광준 본인**: 사업자 세금계산서 발행 → 원천징수(3.3%) 없음
  - **크루원(세컨/서드/막내)에게 지급 시**: 3.3% 원천징수 후 지급
  - 프로젝트 단위가 아닌 "날짜 × 일당" 단순 구조
  - **진행비 포함 정산**: 주차비, 주유비 등 현장 지출을 나중에 묶어서 거래처에 계산서 발행
- **프리랜서 계약**: 프로젝트 단위로 계약 후 납품. 선금/잔금 분할 가능.
  - 원천징수 3.3% 또는 세금계산서 발행
- **공동 사진 프로젝트**: 부부 공동으로 진행. 수익 배분 또는 통합 관리.

## 1. 아키텍처 및 구현 순서 제안

시스템의 가장 뼈대가 되는 '거래처'와 '프로젝트'를 먼저 정의한 뒤, 그 위에 파이프라인과 지출 내역을 붙여나가는 상향식(Bottom-up) 방식을 제안합니다.

- **Step 1: 거래처(Clients) 및 프로젝트(Projects) 기초 설계**
  - `clients` 테이블 생성 (업체 데이터베이스화)
  - `projects` 테이블 생성 (진행하는 건별 프로젝트)
  - 신규 프로젝트 생성 및 관리 UI

- **Step 2: 수입 파이프라인 (Income Pipeline) 및 칸반 보드 구축 [★ 핵심]**
  - 다중 수입 처리 (선금, 중도금, 잔금 등 분할 결제 지원)
  - 퍼스트 출장 일당 전용 빠른 입력 UI
  - 상태 관리 (진행중 → 작업완료 → 입금전 → 정산완료)
  - Trello 스타일의 칸반 보드 시각화 UI 추가

- **Step 3: 계좌 실거래내역(Transactions) 통합 및 지출 태깅**
  - 실제 은행 계좌로 입금된 내역(`transactions`)과 수입 파이프라인 내역 매칭
  - 가계부 지출 내역 중 '사업비'를 특정 프로젝트(`project_id`)로 연결하여 프로젝트별 원가/마진 계산

- **Step 4: 파생 비즈니스 로직 (견적서, 세금, 대시보드)**
  - 견적서 자동 생성 및 PDF 출력
  - 인건비 외주 원천징수(3.3%) 계산
  - ERP 전용 대시보드 완성

## 2. 주요 데이터 모델 요구사항 정리

### 2.0. 자사 정보 (My Business Profiles)

**[수정]** 단일 회사가 아닌 **3개 사업 주체**를 각각 관리.

- **목적:** 견적서 자동 완성, 포트폴리오/레퍼런스 관리, 세금계산서 발행.
- **필요 속성 (주체별):**
  - 사업주 이름 및 사업자번호
  - 상호 (없을 경우 이름으로 대체)
  - 주소, 입금 계좌정보
  - 기본 포트폴리오 링크 / 회사소개서
  - **견적서 포함 옵션:** 포트폴리오 첨부 여부 체크박스

**예시:**

| 프로필 | 상호 | 담당 |
|--------|------|------|

| 의영 사업자 | (개인) 송의영 | 그래픽/공간 디자인 |
| 공동 사진사업 | (추후 결정) | 사진 촬영 프로젝트 |

### 2.1. 거래처 (Client)

- **목적:** 견적서 발행, 누적 거래액 관리, 정보 보관용 정규화 데이터베이스.
- **필요 속성:**
  - 업체명 (필수)
  - **기본 시드 데이터:** 스튜디오리프, 기브온, 그립두드, 한국 다이이치산쿄 등
  - 사업자등록번호
  - 사업자등록증/관련 파일 첨부 (안전한 스토리지 링크)
- **연결 속성 (1:N 담당자):** 한 업체 안에 여러 부서/담당자 — 노션 스타일의 하위 테이블이나 유연한 JSON 필드로 다중 연락망 관리.
- **[LTV 및 인사이트 추적]:** 업체별 누적 매출액, 총 프로젝트 횟수, 최근 거래일 표시.

---

### 2.2. 프로젝트 (Project)

- **목적:** 하나의 계약(Job) 단위. 특정 프로젝트 안에서 여러 번의 수입(선금/잔금)과 지출이 발생함.
- **필수 속성:**
  - 프로젝트명
  - **사업 주체 (business_owner):** 광준 / 의영 / 공동 — 통계 및 세무 분리를 위한 필수 값
  - **수입 유형 (income_type):**
    - `freelance` — 프리랜서 계약 (선금/잔금 분할 가능)
    - `daily_rate` — 퍼스트 일당 (광준 전용, 날짜 × 단가 구조)
    - `photo_project` — 공동 사진 프로젝트
  - 연결된 거래처 (Client ID)
  - **[카테고리 다중 태깅]:** 아래 표준 태그 목록 사용

**[수정] 프로젝트 카테고리 표준 태그 목록:**

| 태그 | 담당 | 설명 |
|------|------|------|
| `영상조명` | 광준 | 영상 촬영 현장 조명 (CF, 드라마, 광고 등) |
| `영상스케치` | 광준 | 영상 스케치/촬영 (겸임) |
| `사진조명` | 광준 | 사진 촬영 현장 조명 |
| `조명_퍼스트` | 광준 | 하루 단위 현장 스태프 (세금계산서 발행) |
| `조명_프리랜서` | 광준 | 계약 단위 조명 프로젝트 |
| `그래픽디자인` | 의영 | 브랜드, 인쇄, 광고 디자인 |
| `공간디자인` | 의영 | 공간/인테리어 관련 디자인 |
| `모션그래픽` | 의영 | 영상/모션 그래픽 |
| `사진촬영_공동` | 공동 | 부부 공동 사진 프로젝트 |
| `중계` | 광준 | 스포츠/행사 중계 조명 |

  - **[시간/일당 마진 추적]:** 총 소요 일수 입력 → 최종 매출-지출(원가) 후 일당/시급 대비 순수익 산출.

- **지출 관리:** 가계부에서 '사업비' 분류 시 현재 [진행중]인 프로젝트만 표시하여 태깅 편의성 극대화. 양방향 뷰 제공.

### 2.3. 퍼스트 일당 기록 (Daily Rate Log) [광준 전용, NEW]

광준의 "퍼스트" 출장은 프로젝트 단위가 아닌 **날짜 단위** 기록이 필요합니다.

- **목적:** 하루 현장 나갈 때 빠르게 기록, 월별/연별 일당 수입 집계.
- **필요 속성:**
  - 날짜
  - 거래처 (Client ID)
  - 현장명/작품명 (예: "현대차 CF", "드라마 OO")
  - 일당 금액
  - **세금계산서 발행** (광준 본인 기본값. 원천징수 없음)
  - 결제일 (실제 입금일)
  - 연결 transaction_id (입금 후 매칭)
  - **진행비 항목 (별도 테이블 or JSONB):** 주차비, 주유비, 톨비 등 현장 지출 목록 → 합산해서 거래처에 계산서 발행
- **빠른 입력 UI:** 모달로 "날짜 + 거래처 + 금액" 3개 기본 입력. 진행비는 선택적으로 추가.
- **월별 요약:** 이번 달 총 출장 일수, 총 수입 (일당 + 진행비 합산).

### 2.3-1. 크루원(인력) 관리 (Crew Management) [NEW]

광준이 퍼스트로서 세컨/서드/막내 등 크루원에게 인건비를 지급하고, 이를 거래처에 포함해서 계산서 발행하는 흐름 관리.

- **목적:** 크루원 인건비 지급 기록 + 3.3% 원천징수 계산 + 거래처 정산 계산서 자동 합산.
- **흐름:**
  ```
  거래처(기브온/그립두드) → 광준에게 총액 지급
  광준 → 크루원에게 지급 (3.3% 원천징수 후)
  광준 → 거래처에 계산서 발행 (일당 + 크루 인건비 + 진행비 합산)
  ```
- **필요 속성 (크루원 지급 기록):**
  - 날짜, 거래처/현장
  - 크루원 이름, 역할 (세컨/서드/막내)
  - 지급 금액 (세전), 원천징수 금액 (3.3%), 실수령액
  - 계좌 정보, 지급 완료 여부
- **정산 계산서 자동 합산:** 현장 1건에 대해 (광준 일당 + 크루 인건비 합계 + 진행비) → 거래처 청구 총액 자동 계산
- **기존 데이터 참고:** `일해주신분들 원천` 시트 (2025 가계부 파일 내) 양식 참고

### 2.4. 공동 사진사업 (Joint Photo Business) [NEW]

부부가 함께 운영하는 사진 촬영 사업 전용 섹션.

- **목적:** 공동 수익 관리, 사진 프로젝트별 수입/지출 분리 관리.
- **상호**: **스튜디오XX** (이름 미확정) — 견적서/계산서 발행 시 이 상호 사용
- **운영 구조:**
  - 광준: 사진 촬영 + 조명 디렉팅
  - 의영: 후반 편집/보정, 디자인 (앨범, 패키지 등)
  - 수익: 공동 계좌로 입금 후 분배 또는 통합 관리
- **필요 기능:**
  - 사진 프로젝트 등록 (웨딩, 프로필, 상품, 가족사진 등 유형 태그)
  - 프로젝트별 견적서/계약서 발행 (공동 명의)
  - 지출 항목: 장비 렌탈, 스튜디오 대여, 인화/앨범 제작비 등
  - 수익 분배 뷰: 총 수입에서 공동 지출 제외 후 순이익 표시
- **사진 유형 태그 (하위 카테고리):**
  - `웨딩` / `스드메` / `프로필` / `상품사진` / `가족사진` / `기업행사` / `기타`

### 2.5. 수입 파이프라인 (Income & Pipeline)

- **목적:** 돈을 언제, 얼마나 받을지 예상하고 추적하며 각 흐름을 관리.
- **관리 대상(카드 단위):** "프로젝트 1개당 카드 1개".
  - 분할 결제는 카드 내부에 **체크리스트** 형태 (선금 / 잔금).
  - 퍼스트 일당은 별도 "일당 로그" 뷰에서 관리 (칸반 보드 오염 방지).

- **칸반 카드 표면 표시 정보 (UI 기준):**
  1. 프로젝트명 (가장 크게)
  2. 마감일
  3. 현재 진행 상태 (뱃지)
  4. 사업 주체 아이콘 (광준 🟦 / 의영 🟩 / 공동 🟪)
  - (+선택 시 청구 금액, 거래처명 등)

- **칸반 보드 상태(Status) 파이프라인:**
  1. **의뢰중:** 문의 수기 접수 및 협의 단계
  2. **작업중:** 제안 수락 및 실제 작업 진행
  3. *(분기)* **취소/보류:** 드롭된 건들에 대한 아카이브
  4. **작업완료:** 납품은 했으나 고객 검수 중
  5. **수정완료:** 최종 완료 (청구 직전 단계)
  6. **입금대기 (청구완료):** 견적/세금계산서 발행 완료, 입금 대기
  7. **입금/정산완료:** 실제 계좌에 꽂히고 매칭 처리됨
  8. **포스팅완료:** SNS 업로드 등 사후 작업까지 끝난 완전 종료 상태

- **기존 데이터 마이그레이션 (25년):** 기존 수입 35건은 칸반에 올리지 않고 히스토리 리스트로만 보관. **칸반 보드는 2026년 진행 건부터 시작.**

## 3. 대시보드 및 재무/세무 필수 뷰

### 3.1. 개인 가계부 (Personal) 추가 위젯

- **지출 배틀 요약표:** 이번 주/이번 달 기준으로 '광준' vs '의영' 중 누가 돈을 더 많이 썼는지 직관적으로 표시.

### 3.2. 사업 ERP (Business) 전용 화면

- **현금 흐름 캘린더 (Cash Flow):** 입금 예정일과 사업용 고정비(월세, 구독료 등) 출금일을 매핑, 예상 가용 현금액(런웨이) 시각화.

- **원천징수 / 세금 알림:**
  - 퍼스트 일당 원천징수(3.3%) 누적액 자동 계산
  - 프리랜서 계약 원천징수 여부 체크
  - 종합소득세 / 부가가치세 납부 예상액 경고(리저브)

- **시급/일당 효율 차트:**
  - 프로젝트 카테고리 태그 + 소요 일수 기반
  - "이번 달 시간 대비 가장 이득을 많이 본 작업 유형" 요약 차트
  - 퍼스트 일당 vs 프리랜서 계약 수익 비교

- **공동 사진사업 요약 위젯:** 월별 사진 프로젝트 건수, 총 수입, 순이익 표시.

### 3.3. 사업 주체별 분리 뷰

| 뷰 | 내용 |
|----|------|
| 광준 사업 | 영상/사진 조명 + 퍼스트 일당 수입 통합 |
| 의영 사업 | 그래픽/공간 디자인 수입 통합 |
| 공동 사진사업 | 사진 프로젝트 수입/지출/순이익 |
| 전체 통합 | 세 주체 합산 총 사업 현황 |

## 4. 기능 단위 구현 계획 (Implementation Specs)

### 4.1. Database Schema Additions (Supabase SQL Migrations)

**1) 거래처 테이블 (`clients`)**
- `id`: UUID (Primary Key)
- `name`: TEXT (예: 스튜디오리프, 기브온)
- `business_number`: TEXT (NULLABLE)
- `files`: JSONB (사업자등록증 등 파일 URL 목록)
- `contacts`: JSONB (담당자 연락망 정보 배열: 이름, 부서, 직급, 연락처, 이메일)
- `created_at`: TIMESTAMPTZ

**2) 프로젝트 테이블 (`projects`)**
- `id`: UUID (Primary Key)
- `name`: TEXT
- `client_id`: UUID (Foreign Key to `clients`)
- `business_owner`: TEXT (ENUM: `'kwangjun'`, `'euiyoung'`, `'joint'`)
- `income_type`: TEXT (ENUM: `'freelance'`, `'daily_rate'`, `'photo_project'`)
- `categories`: TEXT[] (다중 태그 - 영상조명, 사진조명, 그래픽디자인 등)
- `duration_days`: NUMERIC (투입 소요 일수/시간 계산용, NULLABLE)
- `start_date`, `end_date`: DATE
- `created_at`: TIMESTAMPTZ

**3) 퍼스트 일당 로그 테이블 (`daily_rate_logs`)** [NEW]
- `id`: UUID (Primary Key)
- `user_id`: UUID (Foreign Key)
- `client_id`: UUID (Foreign Key to `clients`)
- `work_date`: DATE (현장 날짜)
- `site_name`: TEXT (현장명/작품명)
- `amount_gross`: NUMERIC (세전 금액)
- `withholding_rate`: NUMERIC (DEFAULT 3.3)
- `amount_net`: NUMERIC (세후 금액, 자동 계산)
- `payment_date`: DATE (실제 입금일, NULLABLE)
- `matched_transaction_id`: UUID (NULLABLE, 입금 매칭 후 채움)
- `created_at`: TIMESTAMPTZ

**4) 크루원 지급 기록 테이블 (`crew_payments`)** [NEW]
- `id`: UUID (Primary Key)
- `daily_rate_log_id`: UUID (FK → `daily_rate_logs`, 어느 현장인지 연결)
- `crew_name`: TEXT (이름)
- `role`: TEXT (세컨 / 서드 / 막내 / 기타)
- `amount_gross`: NUMERIC (세전)
- `withholding_rate`: NUMERIC (DEFAULT 3.3)
- `amount_net`: NUMERIC (실지급액, 자동 계산)
- `account_info`: TEXT (계좌번호)
- `paid`: BOOLEAN (지급 완료 여부)
- `paid_date`: DATE

**4-1) 진행비 테이블 (`site_expenses`)** [NEW]
- `id`: UUID (Primary Key)
- `daily_rate_log_id`: UUID (FK → `daily_rate_logs`)
- `category`: TEXT (주차비 / 주유비 / 톨비 / 식비 / 기타)
- `amount`: NUMERIC
- `memo`: TEXT
- `receipt_url`: TEXT (영수증 첨부, NULLABLE)
- `included_in_invoice`: BOOLEAN (계산서에 포함 여부)

**5) 수입 파이프라인 테이블 (`project_incomes`)**
- `id`: UUID (Primary Key)
- `project_id`: UUID (Foreign Key to `projects`)
- `title`: TEXT (예: '선금', '잔금', '계약금')
- `amount`: NUMERIC
- `expected_date`: DATE (마감일/입금예정일)
- `status`: TEXT (의뢰중 / 작업중 / 보류취소 / 작업완료 / 수정완료 / 입금대기 / 입금완료 / 포스팅완료)
- `matched_transaction_id`: UUID (실제 입금 시 `transactions` 테이블 내역과 1:1 매칭)

**5) 기존 지출 연동 (`transactions` 테이블 수정)**
- `project_id`: UUID (Foreign Key to `projects`, 기본값 NULL) - 가계부 지출 시 특정 사업 프로젝트로 태깅.

### 4.2. Frontend UI/UX 로직

1. **마이 비즈니스 설정 페이지**: 3개 사업 주체 각각 정보 및 견적서 옵션 관리
2. **거래처 관리 뷰**: 하위 동적 연락처 폼, LTV 정보 연동
3. **프로젝트 칸반 보드**: 상태별 Trello 뷰, 사업 주체 필터, 카드 상세 모달
4. **퍼스트 일당 빠른 입력**: 모달 3단계 입력 (날짜 + 거래처 + 금액), 월별 요약
5. **공동 사진사업 전용 뷰**: 사진 유형별 프로젝트 관리, 수익 분배 표시
6. **인사이트 대시보드**: 개인 지출 대결, 사업 현금 흐름 캘린더, 시간당 마진 효율, 사업 주체별 수입 비교
