## 🔑 **주요 설계 변경사항**

### **반품/취소 처리 (핵심)**

```
원래 설계:
  - 상품 분해만 고려
  - 반품/취소 미처리

개선된 설계:
  - 거래명세표 파싱
  - 취소 영수증 자동 파싱 ⭐
  - 반품 상품 자동 식별 ⭐
  - Refund 타입 추가 ⭐
  - 최종 금액 자동 검증 ⭐
```

### **UI 단계 확장**

```
5-Step → 7-Step 다이얼로그

추가된 Step:
  - Step 3: 반품/취소 확인 [취소 영수증 업로드]
  - Step 4: 수령/반품 상태 확인 [상태 선택 + 자동 감지]

결과:
  - 사용자가 한 번의 업로드로 모든 정보 입력
  - 시스템이 자동으로 반품/수령 구분
  - 최종 금액 자동 검증
```

### **DB 스키마 추가**

```sql
transactions 테이블:
  - type (VARCHAR): 'expense', 'refund' 등
  - is_refund (BOOLEAN): 반품 여부
  - refund_reason (VARCHAR): 반품 사유
  - refund_date (TIMESTAMP): 반품 처리 일자

transaction_breakdowns 테이블:
  - refund_amount (NUMERIC): 환급 금액
  - refund_reason (VARCHAR): 환급 사유
```

## 📞 **질문사항**

설계 중 결정된 사항:

| 결정 | 선택 | 근거 |
|------|------|------|
| **반품 처리 방식** | 보정 거래 (Strategy C) | 자동화 수준이 높고 정확도가 우수 |
| **취소 영수증 입력** | 선택 사항 | 없으면 그냥 분해, 있으면 자동 감지 |
| **퍼지 매칭** | 80% 이상 유사도 | 오타/줄임말 대응, 수동 선택 Fallback |
| **Step 추가** | +2 Step (7-Step) | 반품 확인이 필수 → 별도 단계 필요 |

```
사용자 액션                        시스템 액션
═══════════════════════════════════════════════════

1. 쿠팡 거래명세표 업로드
   (Coupang거래명세표.pdf)
                                  → PDF 파싱
                                  → 상품 4개 추출

2. [선택] 취소 영수증 업로드
   (있으면 업로드, 없으면 스킵)
                                  → OCR 파싱
                                  → 반품 항목 추출
                                  → 상품명 퍼지 매칭

3. 수령/반품 상태 확인
   (자동 감지 + 수동 확인)
                                  → 각 상품 상태 확정
                                  → 최종 금액 검증

4. 카테고리 분류
   (수령 상품만)
                                  → Smart Tagging 제안

5. 미리보기
   (최종 확인)
                                  → 반품은 빨간색 표시

6. 완료
   (거래 생성)
                                  → 원본 거래 숨김
                                  → 분해된 거래 생성
                                  → 가계부 업데이트

결과:
  ├─ "오넛티" 11,900원 [생활비]
  ├─ "모나리자" 4,710원 [생활비]
  ├─ 🔴 "[반품환급] 칫솔" -18,500원
  └─ "크리넥스" 6,300원 [생활비]

최종 합계: 38,460원 ✓
```
