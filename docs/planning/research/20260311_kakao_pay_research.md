# 🕵️ 카카오페이 내역 관리 연구 (Kakao Pay Integration)

## 1. 현황 및 문제점
- **불투명한 요약 내역**: 은행/카드사 내역에서 실가맹점 확인 불가 ('카카오페이'로만 표시).
- **다양한 적요 패턴**: '카카오페이' 외에도 '주식회사 카카오', '카카오T', '카카오스타일' 등 다양한 적요가 혼재함.
- **추출 제한**: 앱 내 다운로드는 50건 제한.
- **데이터 중복 위험**: 충전(은행)과 결제(페이)가 모두 지출로 잡히는 현상.

## 2. 조사 결과 (해결 방안)

### A. 대량 내역 추출 (확인 완료)
- **카카오페이 고객센터(챗봇)** > **증명서 발급** > **거래내역서 발송** 기능을 통해 수령한 엑셀 파일 분석 결과:
  - **헤더**: `거래일시`, `거래구분`, `거래금액`, `거래 후 잔액`, `은행`, `계좌 정보 / 결제 정보`
  - **가맹점 정보**: '계좌 정보 / 결제 정보' 컬럼에 실제 가맹점 또는 수취인명이 포함됨을 확인.

### B. 데이터 매칭 및 중복 방지 (핵심 전략)
'카카오페이 머니'를 하나의 독립된 **자산(Asset)**으로 봅니다.

| 순서 | 발생 내역 | 가계부 처리 (Type) | 자산 흐름 |
| :--- | :--- | :--- | :--- |
| **1** | 은행에서 1만원 충전 (`부족분충전`) | **이체 (Transfer)** | IBK계좌 → 카카오페이 머니 |
| **2** | 페이머니로 7천원 결제 (`[-] 결제`) | **지출 (Expense)** | 카카오페이 머니 → 가맹점 |
| **3** | 친구에게 3천원 송금 (`[-] 송금`) | **지출 (Expense)** | 카카오페이 머니 → 지인 |

- **은행 내역 임포트 시**: '카카오페이' 적요는 무조건 **'이체'**로 분류되도록 파서 고도화.
- **페이 내역 임포트 시**: `[+] 부족분충전`은 **'이체'**, `[-] 결제/송금`은 **'지출'**로 분류.
- **결과**: 충전 시점에는 지출이 발생하지 않고, **실제 돈을 쓴 시점에만 지출**로 기록되어 데이터가 정확해집니다.

## 3. 거래 유형별 분류 (Transaction Types)

카카오페이 거래내역서의 '거래구분' 컬럼에 따른 분류 규칙:

| 거래구분 | 의미 | 가계부 처리 | 비고 |
|---------|------|----------|------|
| `[+] 부족분충전` | 은행 → 카카오페이 | **Transfer** | Asset 흐름: IBK → 카카오페이 머니 |
| `[-] 결제` | 카카오페이 → 가맹점 | **Expense** | Asset 흐름: 카카오페이 머니 → 실제 가맹점 |
| `[-] 송금` | 카카오페이 → 개인 | **Expense** | 지인에게 송금 |
| `[+] 환급` | 가맹점 → 카카오페이 | **Refund** | 반품/취소 시 환급 |
| `[-] 수수료` | 카카오페이 수수료 | **Expense** | 카테고리: 기타비용 > 지급수수료 |
| `[+] 포인트/혜택` | 카카오페이 리워드 | **Income** | 소액 보너스 |

### C. 은행 내역 적요 패턴 (필터링 키워드)
DB 분석 결과, 카카오페이 내역서와 매칭될 수 있는 원본 은행 거래의 적요 패턴은 다음과 같습니다:
- **페이/송금 계열**: `카카오페이`, `선물하기_카카오페`, `카카오(선물하기)`
- **모빌리티 계열**: `카카오T`, `카카오모빌리티`, `카카오_택시`
- **커머스/기타 계열**: `주식회사 카카오`, `카카오스타일`, `카카오페이손해보험`, `카카오(상품권)`

### D. 타 간편결제/PG 결제 내역 (추가 조사 필요)
DB 분석 결과, 카카오페이 외에도 상세 내역 분해가 필요할 수 있는 간편결제/PG사 내역이 다수 존재합니다:
- **네이버페이**: `네이버페이 네이버`, `네이버파이낸셜(주 네이버` (총 10건 이상)
- **당근페이**: `당근페이` (4건)
- **토스페이**: `토스페이` (1건)
- 기타: `구글페이먼트코리아`, `다날` 등

**제언**: 카카오페이 매퍼의 로직을 범용화하여, 추후 네이버페이, 당근페이 등의 거래내역서(Excel/CSV)를 업로드했을 때도 동일한 방식으로 원본 결제를 파악하고 매칭할 수 있도록 확장하는 것을 고려해야 합니다.

## 4. 가맹점 정보 추출 (Merchant Mapping)

### 데이터 출처
카카오페이 거래내역서의 **"계좌정보 / 결제정보" 컬럼**에서:
- **직접 결제**: "쿠팡", "배민", "컬리" 등 실제 가맹점명
- **송금/이체**: "홍길동 → 계좌" 형식
- **출금**: "출금" 또는 은행명

### 자동 매칭 프로세스
```
Step 1: 은행 카드사 내역 임포트
  "카카오페이" 10,000원 (2026-03-11)

Step 2: 카카오페이 내역서 파일 업로드
  ├─ 2026-03-11 쿠팡 7,000원 ([-] 결제)
  └─ 2026-03-11 배민 3,000원 ([-] 결제)

Step 3: 자동 매칭 (거래일 & 금액)
  "카카오페이" 10,000원
    ↓ [일치 확인: 합계 10,000원]
  ├─ "쿠팡" 7,000원
  └─ "배민" 3,000원

Step 4: 거래 분해 및 카테고리 할당
  ├─ [카카오페이] 쿠팡 7,000원 → 물품구입비
  └─ [카카오페이] 배민 3,000원 → 식비 > 배달
```

### 설명 형식
```
"[카카오페이] 쿠팡"  ← 출처 명확화 (카카오페이 중개)
```

## 5. 쿠팡과의 차이점 (vs. Coupang Card)

| 구분 | 쿠팡 카드 | 카카오페이 |
|------|---------|----------|
| **카드사 내역** | "쿠팡" → 가맹점 명확 | "카카오페이" → 불명확 |
| **추가 정보** | 카드사 앱에서 일부 상세 정보 | 카카오페이 거래내역서 필수 |
| **분류 난도** | 낮음 (직접 가맹점 인식) | 중간 (가맹점 매칭 필요) |
| **분해 필요** | **YES** (다중 상품 1회 결제) | NO (카카오페이 내역서가 이미 분해) |
| **사용자 개입** | 상세정보 업로드 필수 | 거래내역서 자동 다운로드 가능 |
| **자동화 가능성** | 중간 (상품명 기반 카테고리) | 높음 (가맹점명 매칭) |

**핵심 차이**: 쿠팡은 "1개 결제를 N개로 분해", 카카오페이는 "N개 거래를 1개 결제로 통합"

## 6. 이중 지출 방지 (Double Count Prevention)

### 패턴
```
은행 입출금 내역:
  2026-03-11 쿠팡 -50,000원 (직접 카드 결제 또는 카카오페이)

쿠팡 분해 (별도 도구):
  ├─ 노트북: -30,000원 [사업비]
  ├─ 우유: -5,000원 [생활비]
  └─ 선물: -15,000원 [경조/선물]

문제: 원본 50,000원 + 분해 50,000원 = 100,000원 (중복!)
```

### 해결책
```
1. 쿠팡 분해 도구에서 원본 거래는 "숨김" 처리
2. 분해된 N개 항목만 가계부에 노출
3. 통계 계산 시 excluded_from_personal 필터 적용
   (사업비인 "노트북" 항목은 개인 순자산 통계에서 제외)
```

## 7. 구현 세부 계획 (Next Steps)

### Phase 1: 기반 (Foundation)
1. **Asset 추가**: `assets_liabilities` 테이블에 "카카오페이 머니" 생성
2. **DB 스키마**:
   - `transaction_breakdowns` 테이블 (쿠팡 분해 추적)
   - `kakao_pay_mappings` 테이블 (카카오페이 매칭 추적)
   - `transactions.excluded_from_personal` 컬럼 추가

### Phase 2: Tool B (카카오페이 가맹점 매퍼) - **우선 구현**
1. **CSV 파서**: 카카오페이 거래내역서 자동 파싱
2. **매칭 엔진**: 거래일 & 금액 기반 자동 매칭
3. **분류 로직**: 거래 유형별 분류 (충전 = Transfer, 결제 = Expense)
4. **Smart Tagging 연계**: 가맹점명 → 카테고리 자동 제안
5. **Asset 흐름 기록**: 충전(이체) vs 결제(지출) 분리
6. **UI 개발**: `/transactions/kakao-pay-matcher` 페이지

### Phase 3: Tool A (쿠팡 상세 내역 분해기) - **2차 구현**
1. **파서 개발**: 쿠팡 주문서 파싱 (이메일, CSV, 스크린샷)
2. **분해 로직**: 상품별 금액 분해 & 검증
3. **카테고리 매칭**: Smart Tagging과 연계
4. **Double Count 처리**: `excluded_from_personal` 플래그
5. **UI 개발**: `/transactions/coupang-breakdown` 페이지

### Phase 4: 통합 & 최적화
1. **혼합 시나리오**: 카카오페이로 쿠팡 결제한 경우 테스트
2. **통계 필터링**: 개인 지출 vs 비즈니스 비용 분리
3. **가계부 UI**: 분해된 항목 시각적 구분 (원본 거래 숨김, 분해항목 표시)
