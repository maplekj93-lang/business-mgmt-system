## 🎯 **핵심 개념 5분 요약**

### **문제 1: 카카오페이**
```
카드사: "카카오페이" 10,000원
↓ [불명확! 실제가 뭔지 모름]
카카오페이: 쿠팡 7,000원 + 배민 3,000원

→ 해결책: "카카오페이 매퍼" 도구
  자동 매칭으로 쿠팡/배민 분리
```

### **문제 2: 쿠팡**
```
카드사: "쿠팡" 50,000원 (1회 결제)
↓ [여러 상품이 섞여 있음]
실제: 노트북 30,000원 (사업용)
    + 우유 5,000원 (생활비)
    + 선물 15,000원 (경조/선물)

→ 해결책: "쿠팡 분해기" 도구
  상세정보 업로드해서 분리
  + 사업용은 개인 통계에서 제외
```

### **핵심: 두 도구는 정반대**
```
쿠팡  = "1개 → N개" (분해)
카카오= "N개 → 1개" (통합/매칭)

따라서 상황에 따라 필요한 도구가 다름!
```

### **시간순서**
```
Phase 1: 기반 작업 (Foundation)     ← 먼저
  ├─ Asset 추가
  ├─ DB 테이블 생성
  └─ 컬럼 추가

Phase 2: 카카오페이 매퍼 (자동, 쉬움) ← 다음
  └─ 거래내역서 자동 매칭

Phase 3: 쿠팡 분해기 (수동, 어려움)   ← 마지막
  └─ 상세정보 수동 입력

Phase 4: 통합 & 최적화               ← 끝
```

## 📞 **질문 & 답변**

**Q: 왜 두 개의 도구가 필요한가?**
```
A: 문제의 성격이 정반대이기 때문
   - 쿠팡: 1개를 N개로 "분해" (분해기 필요)
   - 카카오: N개를 1개로 "통합" (매퍼 필요)
```

**Q: 어느 것부터 구현해야 하나?**
```
A: 카카오페이 매퍼부터 (Phase 2)
   이유:
   - 자동화 수준이 높음 (자동 매칭)
   - 사용자가 이미 필요로 함
   - 쿠팡 분해기보다 개발이 쉬움
```

**Q: 카카오페이와 쿠팡을 동시에 사용하는 경우는?**
```
A: 카카오페이 매퍼 + 쿠팡 분해기 차례대로
   1단계: 카카오페이 매퍼
          "카카오페이" → "쿠팡" (가맹점 명확화)

   2단계: 쿠팡 분해기 (선택)
          "쿠팡" → 상품별 분해
          (상세정보 있을 때만)
```

**Q: 개발 시간 예상은?**
```
A: Phase별로 다름
   - Phase 1: 1-2일 (기반)
   - Phase 2: 4-5일 (카카오페이 매퍼)
   - Phase 3: 5-7일 (쿠팡 분해기)
   - Phase 4: 2-3일 (통합 & 최적화)

   합계: 약 2-3주
```
