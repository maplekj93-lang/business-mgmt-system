## 🔴 MUST-HAVE: 없으면 실운영 불가한 기능들

### M-1. 종합소득세(종소세) 예비비 격리 박스
**왜 필수인가:** 5월 종소세 신고 시 자금 부족 → 실제 현금 위기 발생 가능.
부가세(10%)는 구현됐지만 종소세(15~20%)가 없으면 세무 충격을 맞을 수 있음.

- **로직:** 매 입금 시 부가세 10% 격리 후 → 남은 금액의 15~20% 추가 격리
- **UI:** 대시보드에 "세금 파우치 2개" — ① 부가세 박스 ② 종소세 박스 색상 분리 표시
- **설정:** 예상 소득 구간별 종소세율 커스텀 (사용자가 직접 %)
- **한국 세율 기준:** 소득 1,200~4,600만원 구간 15%, 4,600만원 초과 시 24%

```
[입금 1,000,000원]
  → 부가세 격리: -90,909원 (VAT 박스)
  → 종소세 격리: -136,364원 (소득세 박스, 15% 가정)
  → 실질 가용: 772,727원
```

---

### M-2. 3개월 생활비 버퍼 (Safety Net) 게이지
**왜 필수인가:** 프리랜서 수입은 불규칙함. 안전망 없이 쓰다 보면 마른 달에 패닉.
모든 프리랜서 금융 전문가들이 "최소 3개월 버퍼"를 필수로 권장.

- **로직:** 사용자가 월 고정 생활비를 설정 → 3배 목표금액 자동 계산
- **UI:** 대시보드 상단 게이지 바 (0%~100%)
  - 0~50%: 🔴 위험 (추가 지출 경고)
  - 50~80%: 🟡 주의 (현재 유지)
  - 80~100%: 🟢 안전
- **연동:** 이 게이지가 100% 미만이면 "성과급 지출" 추천 알림 비활성화

---

### M-3. 이중 지출 차단 로직 (Double-Count Prevention)
**왜 필수인가:** 현재 가계부와 사업비에 같은 지출이 중복 계상될 경우 → 순자산 통계가 틀림.

- **로직:** 지출을 "사업비"로 태깅하는 순간 가계부 지출 총액에서 자동 제거
- **DB:** `transactions.allocation_status = 'business'` 변경 시 → 가계부 집계에서 exclude
- **UI:** 태깅 시 "이 지출이 가계부에서 사업비로 이동됩니다" 확인 토스트
- **색상:** 사업비로 이동된 항목은 가계부 내역에서 별도 색상(회색/취소선)으로 표시

---

### M-4. 크루 페이 은행 교차 검증 (Bank Sync Verifier)
**왜 필수인가:** 앱에서 "지급 완료" 버튼만 누르고 실제 송금을 잊는 케이스 방지.
세무 감사 시 지급 증빙 불일치 → 법적 문제 가능성.

- **방식:** 기업은행/카카오뱅크 엑셀 출금 내역 업로드 → 앱 기록과 1:1 매칭
- **매칭 기준:** 날짜 ±1일, 금액 일치, 이체 메모(이름) 포함 여부
- **결과:** ✅ 매칭됨 / ❌ 앱에만 있고 통장엔 없음 / ⚠️ 통장엔 있는데 앱에 기록 없음
- **출력:** 불일치 목록 → 담당자 확인 후 수동 처리

---

### M-5. 가상 월급 이체 로직 (Virtual Salary Transfer)
**왜 필수인가:** 사업 통장과 가계 통장이 분리되어 있을 때, 월급처럼 이체하지 않으면
"사업이 잘 되는데 생활비가 왜 없지?" 혼란 발생.

- **설정:** 매월 N일, M원을 사업 계좌 → 가계 계좌로 "가상 월급" 이체 기록
- **로직:** 이 이체는 수입도 지출도 아닌 내부이체(Transfer)로 처리 → 중복 계산 차단
- **대시보드:** "이번 달 가상 월급: 지급됨 ✅ / 미지급 ⏳"

---

### M-6. 멀티 카드/사용자 레이어 분리
**왜 필수인가:** 부부가 카드를 함께 쓰면서 아내분 지출을 남편 가계부에 통합하면 → 분석이 무의미해짐.

- **현재 상태:** `owner_type` 필드 존재하나 UI에서 명확한 레이어 분리 없음
- **필요:** 각 거래에 "광준 / 의영 / 공동" 태그 → 통계는 각자 분리 or 합산 선택 가능
- **아내 레이어:** 의영 카드 결제 → 별도 색상 강조 → 의영이 직접 디자인 프로젝트 ID로 태깅 가능

---

## 🟡 SHOULD-HAVE: 있으면 업무 효율이 크게 올라가는 기능들

### S-1. 스마트 태깅 라이트 (Smart Tagging Lite)
**전략:** 완전한 AI/ML 대신 **규칙 엔진 + 유사도 매칭**으로 70~80% 자동분류 달성.

**구현 방법 (단계별):**

1. **키워드 룰 엔진 (즉시 구현 가능)**
   - "스타벅스" → ☕ 커피 자동 분류
   - "ADOBE" → 구독/사업비 자동 분류
   - "주유소/GS25/SK에너지" → ⛽ 유류비 사업비 자동 분류
   - 룰 등록 후 이후 동일 가맹점 100% 자동 처리

2. **캘린더 시간 매칭 (가벼운 버전)**
   - 프로젝트에 촬영일 입력 → 당일 ±2시간 이내 카드 결제 → 해당 프로젝트로 자동 추천
   - OCR 대신 "이 결제 현장 식대 맞나요?" 확인 버튼만 추가

3. **가맹점 그룹 분류 (중기 구현)**
   - 과거 분류 패턴 학습: "A 가맹점을 B 카테고리로 15번 분류 → 룰로 등록 제안"
   - QuickBooks의 Smart Rules 방식 (ML 없이 통계 기반)

---

### S-2. 클라이언트 입금 지연 추적 (Payment Lead-Time Tracker)
**왜 중요한가:** "A 업체는 평균 50일 걸린다"는 데이터 → 버퍼 자금 미리 계산 가능.

- **데이터:** [세금계산서 발행일] ~ [입금 확인일] = 리드타임(일수)
- **집계:** 업체별 평균 리드타임 / 최대 리드타임
- **활용:** 새 프로젝트 수주 시 "이 업체는 평균 50일 후 입금 → 버퍼 X만원 필요" 자동 경고

---

### S-3. 세무사 협업 포털 (Accountant Portal Lite)
**왜 중요한가:** 세무사에게 증빙 전달하는 데 매번 수 시간 소요 → 완전 자동화 목표.

- **기능:**
  - 기간별 필터 → 지출 목록 + 카테고리 엑셀 내보내기
  - "증빙 미첨부" 항목 자동 필터링 → ZIP 번들 다운로드
  - 세무사 전용 읽기 전용 접근 링크 (비밀번호 보호)
- **단기 버전:** 엑셀 내보내기만으로도 80% 해결 가능

---

### S-4. 월간 자동 리포트 (Auto Monthly Report)
**왜 중요한가:** 재무 흐름을 월말에 직접 집계하면 시간 소모 + 누락 가능성.

- **내용:** 월 수입 / 지출 / 순수익 / 전월 대비 / TOP5 지출 카테고리
- **사업 트랙:** 프로젝트별 수익성 / 미수금 현황 / 크루 인건비 합계
- **형식:** 앱 내 리포트 뷰 + PDF 내보내기
- **트리거:** 매월 1일 자동 생성 (전월 데이터 기준)

---

### S-5. 바터(Barter) / 크레딧 처리 로직
**왜 중요한가:** 장비 렌탈 ↔ 사무실 월세 상계 처리를 일반 수입/지출로 넣으면
통장 잔고와 앱 데이터가 괴리됨.

- **로직:** 바터 거래는 `transaction_type = 'barter'`로 저장
- **수입 집계 시:** 바터 제외 → 실제 현금 수입만 표시
- **별도 표기:** "[운영비 절감액] 이번 달 350,000원" 카드로 분리 시각화
- **세무:** 바터 거래도 세금계산서 발행 대상이므로 VAT 격리 필요

---

### S-6. 프로젝트 실질 시급 계산 (Real Hourly Rate)
**왜 중요한가:** "어떤 클라이언트가 시간 대비 가장 남는 일인가?" → 포트폴리오 전략 근거.

- **공식:** `(정산금액 - 크루비 - 경비) / (미팅 + 준비 + 현장 + 정산 시간)`
- **입력:** 프로젝트에 "투입 시간" 필드 추가
- **출력:** 프로젝트별 실질 시급 → 클라이언트 수익성 랭킹

---

### S-7. 구독 서비스 사업비 분리 강제 태깅
**왜 중요한가:** Adobe, Apple, Gemini 등을 사업비로 처리해야 매입세액 공제 가능.
현재 개인/사업 구분이 불명확하면 절세 기회 놓침.

- **설정:** 구독 서비스 목록에 "사업용" 토글 추가
- **효과:** 사업용으로 체크된 구독 → 자동으로 사업비 카테고리 + VAT 공제 대상 표시
- **세무:** "매입세액 공제 가능 구독 서비스 합계: 월 N원" 카드 표시

---

## 🟢 NICE-TO-HAVE: 있으면 좋지만 나중에 해도 되는 기능들

### N-1. 성장 모드 대시보드 (Growth Mode)
현장이 없는 날, 대시보드를 "재무 뷰" 대신 "성장 뷰"로 전환.

- 자기계발 진도 % / 투자 시간 누적
- "오늘 3시간 공부 = 미래 가치 N만원" 환산 (설정한 목표 시급 기준)
- 집안일 루틴 체크리스트 (아내가 등록한 미션)
- 완료 시 긍정 피드백 메시지

**구현 복잡도:** 중간. 별도 라우트(`/growth`) + 자기계발 DB 테이블 추가 필요.

---

### N-2. 비전 보드 (Dream Goal Visualization)
- 드림카 / 여행지 / 목표 사진 업로드
- 저축 진행도에 따라 이미지가 점점 선명해지는 필터 효과
- "목표까지 N개월 남았습니다" 카운트다운

---

### N-3. 장비 ROI 트래커
- 장비별 [구매가 / 총 렌탈 수익 / 수리비 / 감가상각]
- "이 장비 투자 회수 완료!" 알림
- 6:2:2 지분율 기반 수익 자동 배분

---

### N-4. 홈 앱스 & 부부 보상 시스템
- 아내가 등록한 미션 → 남편이 완료 시 팝업 보상 (맥주 쿠폰, 안마권 등)
- 공동 목표 달성 시 양쪽 폰에 동시 축하 알림
- 집안일 루틴 캘린더 연동

---

### N-5. 클라이언트 CRM Lite
- 클라이언트별 연락처 / 협업 이력 / 선호 스타일 메모
- 오래된 관계 "안부 연락" 알림
- 클라이언트별 총 수익 / 프로젝트 수 통계

---

### N-6. 계약서 간이 템플릿 (Legal Lite)
- 1페이지 표준 계약 템플릿 (정산 기한, 취소 정책, 저작권 귀속)
- 간이 서명 (텍스트 동의)
- 프로젝트에 계약서 PDF 첨부

---

### N-7. 주간 리포트 푸시 알림
- 매주 일요일 저녁 → "이번 주 수입 N원, 지출 N원, 다음 주 크루 페이 N원"
- 다음 주 입금 예정 프로젝트 미리 알림

---

## ⚪ FUTURE: 기술 기반이 갖춰진 후 도입할 기능들

### F-1. OCR 영수증 자동 인식
**조건:** Google Vision API 또는 Naver Clova OCR 연동 필요. 월 사용료 발생.
**기능:** 영수증 사진 → 금액/날짜/가맹점 자동 추출 → 거래 자동 생성

---

### F-2. 구글/애플 캘린더 완전 연동
**조건:** Google Calendar API OAuth 설정, 모바일 앱 또는 PWA 전환 필요.
**기능:** 촬영 일정 자동 감지 → 당일 결제 자동 프로젝트 태깅 추천

---

### F-3. 홈택스 세무 데이터 연동
**조건:** 홈택스 API (공공데이터 포털) 연동 필요. 공인인증서 처리 복잡.
**기능:** 과거 부가세/종소세 신고 내역 자동 불러오기 → 공제 패턴 학습

---

### F-4. 은행 API 실시간 연동 (오픈뱅킹)
**조건:** 금융결제원 오픈뱅킹 API 가입 필요. 법인 요건 확인 필요.
**기능:** 자동 거래 내역 수신 → 실시간 가계부 업데이트 (엑셀 업로드 불필요)

---

### F-5. AI 지출 예측 및 이상 감지
**조건:** 최소 12개월 데이터 축적 후 유의미.
**기능:** "이번 달 식비가 평소보다 30% 높습니다" / "다음 달 예상 지출 N원"

---

### F-6. 라이팅 플롯 & 픽스쳐 라이브러리
**조건:** Canvas 기반 드로잉 라이브러리 (Konva.js 등) 별도 구현 필요.
**기능:** 촬영장 도면 위에 장비 배치 드래그앤드롭, DMX 파라미터 저장, 레시피 자산화

---

### F-7. 생체 데이터 연동 알림 (Bio-Sync)
**조건:** Apple HealthKit / Samsung Health API 연동 필요. 개인정보 동의 처리 복잡.
**기능:** 수면 부족 감지 → 알림 톤 조절, 피로도 높은 날 퀵 모드 자동 전환

---

## 💡 외부 레퍼런스 인사이트 (조사 기반)

글로벌 프리랜서 금융 플랫폼(Xero, FreshBooks, Bloom, QuickBooks)을 분석한 결과, L&D ERP에 아직 없지만 고려할 만한 **추가 아이디어** 3가지:

### 추가 제안 1. 프로젝트 기반 시간 추적 통합
FreshBooks, Bloom에서 공통으로 제공. 프로젝트에 "투입 시간" 입력 → 실질 시급 자동 계산.
→ 현재 프로젝트 엔티티에 `total_hours` 필드만 추가하면 구현 가능.

### 추가 제안 2. 미수금 자동 리마인더 (AR Aging)
세금계산서 발행 후 N일 경과 시 → 앱 알림 or 이메일 초안 자동 생성.
→ 국내 프리랜서들이 가장 시간 낭비하는 "입금 독촉" 자동화.

### 추가 제안 3. 분기별 세금 예납 캘린더
한국 종소세 예납은 11월 (직전 연도 종소세의 50%). 캘린더에 세금 납부일 자동 등록.
→ 세금 납부일 D-30, D-7, D-1 알림 → 자금 준비 여유 확보.
