# 거래내역/사용내역 소유자 배정 로직 설계 방향 (Research)

## 1. 현재 구조 분석 (As-Is)
* **UI 레벨**: `TransactionTable`에 "소유자" 컬럼이 이미 존재하며 `tx.owner` 속성을 참조합니다. 하지만 현재 이 값이 매핑되지 않아 모든 내역이 **'미상'**으로 표시되고 있습니다.
* **DB 모델**: `transactions` (거래내역 및 카드 사용내역 통합 테이블)에는 소유자와 관련된 독립적인 컬럼이 없습니다. 대신 `asset_id`를 통해 `assets` 테이블과 연결되며, `assets` 테이블에 `owner_type`('kwangjun', 'euiyoung', 'joint', 'business', 'other') 속성이 존재합니다.
* **현재 의도된 로직**: 자산을 변경할 때 나오는 안내 팝업("자산과 소유자가 변경되었습니다")의 문구로 보아, 기존의 기획 의도는 **"거래내역의 자산 소유자 = 거래내역의 소유자"**로 간주하는 종속(파생) 방식이었던 것으로 보입니다.

---

## 2. 설계 방향 제안 (To-Be)

각 거래내역/사용내역에 소유자를 정해주기 위해서는 크게 두 가지 아키텍처 방향이 있습니다. 서비스의 복잡도와 유연성 요구사항에 따라 선택하실 수 있습니다.

### 옵션 A: 자산(Asset) 종속 모델 (Views/RPC 연동 방식)
가장 단순하고 즉각적인 해결책으로, 기존 기획 의도에 맞춰 DB 스키마 수정 없이 데이터 조회 시점에 자산의 소유자를 렌더링하는 방식입니다.

* **구현 방식**: 
  * `TransactionHistoryWidget` 및 조회 API에서 `tx.owner` 필드에 매핑된 자산의 `owner_type`을 주입 (`owner: tx.asset?.owner_type`).
* **장점**:
  * 추가적인 DB 마이그레이션(`ALTER TABLE`)이 필요 없습니다.
  * 자산만 매핑해주면 소유자가 자동으로 따라가므로 사용자가 소유자를 일일이 지정할 필요가 없습니다.
* **단점 / 한계**:
  * **(치명적)** 공동 카드(가족 공동 자산)로 개인이 사적으로 지출한 경우, 해당 거래내역 한 건만 특정 개인의 소유로 분리할 수 없습니다.
  * 현금 등 자산이 매핑되지 않은(미지정) 기록의 경우, 소유자를 설정할 수 없습니다.

### 옵션 B: 거래내역(Transaction) 독립 소유자 모델 [💡 추천 방향]
`transactions` 테이블에 직접 `owner` 컬럼을 추가하고, 자산 매핑 시 자동으로 상속받되 수동으로 수정할 수 있는 방식입니다. (가계부를 꼼꼼하게 관리하는 경우 필수적인 구조)

* **구현 방식**:
  1. **DB 변경**: `transactions` 테이블에 `owner_type` 컬럼 추가.
  2. **기본값 로직**: 거래내역에 자산(`asset_id`)을 매핑할 때, 거래내역의 `owner_type`을 자산의 `owner_type`으로 자동 업데이트.
  3. **수동 지정 로직**: `TransactionTable` UI에서 자원(Asset) 변경과 독립적으로 소유자(Owner)만 변경할 수 있는 배지/셀렉터 제공.
* **장점**:
  * **완벽한 유연성**: 아내 명의 카드로 남편의 개인 지출을 하거나, 공동 계좌에서 개인이 비용을 쓴 부분 등 현실적인 엣지 케이스를 완벽히 통제/분리할 수 있습니다.
  * 자산 미지정 내역에 대해서도 소유자 누락 없이 할당 가능합니다.
* **단점**:
  * 약간의 DB 마이그레이션(기존 데이터 업데이트)과 UI/API 변경 작업이 수반됩니다.

---

## 3. 요약 및 권장 사항

가족 공동 생활비 관리와 법인(Business) 지출이 혼재되어 있는 시스템이므로, 장기적으로는 유연성과 데이터의 정확성을 위해 **[옵션 B: 독립 소유자 모델]**을 권장합니다.

## 4. 현재 상황 (2025-03-05 기준)

**✅ 이미 완료된 작업:**
- 2025 가계부 데이터: 1,444건 import 완료 (개인 지출만 포함)
- 자산 DB에 18개 자산 등록 (광준/의영/공동 명의 계좌/카드)
- 미분류 내역: 81건 (자산이 잘못 매핑되거나 빈 내용)

**⚠️ 즉시 할 일 (Phase 1 - 현 가계부 정리):**
1. `transactions` 테이블에 `owner_type` 컬럼 추가 (NOT NULL, DEFAULT 'joint')
2. 기존 1,444건 데이터에 대해 자산의 `owner_type`을 거래내역의 `owner_type`으로 복사
   ```sql
   UPDATE transactions t
   SET owner_type = a.owner_type
   FROM assets a
   WHERE t.asset_id = a.id AND t.asset_id IS NOT NULL;
   ```
3. TransactionTable UI에서 owner 셀 클릭 → 드롭다운으로 수동 변경 가능하도록 개선

**📅 Phase 2 (2026년 데이터 import 시):**
1. 은행/카드 명세서 파일에서 추출한 계좌번호/카드번호 → 자산 자동 매핑
2. 자산의 `owner_type` → 거래내역의 `owner_type` 자동 할당
3. 예시:
   - 파일: `삼성카드_379183****784.xlsx`
   - 자동 인식: 카드번호 = 광준 계정의 삼성카드
   - 결과: 거래 100건 모두 `owner_type = 'kwangjun'`으로 import
