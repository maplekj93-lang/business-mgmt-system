# L&D ERP 구현 현황 최종 분석 리포트 (통합 결산본)

V1.1부터 V2.7까지의 전체 명세서(`L_and_D_ERP_Specification_V2.7.md`) 대비 현재 코드베이스의 **① 완전 누락 기능**과 **② 구현은 되었으나 디테일이 부족한 반쪽짜리 기능**을 총망라한 최종 분석서입니다.

---

## 🟩 1. 대체로 잘 구현된 뼈대 기능 (운영 가능 수준)
- **프로젝트 파이프라인 (Project)**: Pre-Pro-Post 칸반 보드 관리 및 매출 매핑(`match-income`) 기능.
- **크루 일급 대장 (Daily Rate)**: 지급 기록 수정, 과거 기록 조회, 원천징수 계산.
- **부가세 격리 (VAT)**: 부가세 분리 모니터링 카드.
- **고정비 캘린더 (Recurring Expense)**: 달력 화면을 통한 고정 구독 지출 추적.

---

## 🟨 2. 구현은 되었으나 '디테일이 부족한' 반쪽짜리 기능
이 기능들은 작동은 하지만 명세서가 요구하는 깊은 세부 로직이 빠져 있습니다.

1. **가계부 대시보드 (3개월 버퍼 시각화)**
   - **현재:** 좌(가계) 우(사업) 화면 분리와 기초적인 예산/사용액 시각화.
   - **누락 디테일:** 필수 생활비가 모였는지 보여주는 **'3개월 버퍼 (Safety Net) 게이지'** 뷰가 빠져있습니다. (사용자 필수 기능)

2. **세무 격리 (종소세/부가세 파우치) 및 구독료 분리 (필수)**
   - **현재:** 부가세(10%) 계산기만 존재.
   - **누락 디테일:** 부가세 외에 매출의 15~20%를 **'종합소득세 예비비 박스'**로 추가 격리하는 로직이 필수적으로 필요합니다. 또한 고정 지출(어도비 등)을 사업용/개인용으로 엄격하게 색상이나 태그로 강제 분리하는 UX가 미흡합니다.

3. **크루 인건비 은행 교차 검증 (Bank Sync Verifier) (필수)**
   - **현재:** 대장에서 '지급 완료' 버튼 클릭 가능.
   - **누락 디테일:** 사용자가 앱에서 버튼만 누르고 실제 은행 송금을 잊을 경우를 대비해, 엑셀 출금 내역과 앱 상태를 1:1로 비교해 주는 **'통장 대조기'** 로직이 필요합니다.

---

## 🟥 3. 아예 코드로 만들어지지 않은 '완전 누락' 기능
이 기능들은 앱의 진정한 가치인 **AI 자동화 및 워크라이프 밸런스**를 담당하지만, 아직 개발되지 않았습니다.

1. **지능형 스마트 태깅 & 캘린더 연동 (효율적 구현 목표)**
   - 구글/애플 캘린더 일정을 불러와서, 카드 결제 시간/위치와 교차 검증해 "이 식대는 현장 소모품입니다"라고 **자동 추천 및 분류**해 주는 지능형 알고리즘 로직이 전무합니다. (※ 사용자 요청: 개발 시간이 너무 오래 걸리지 않는 선에서 효율적/가벼운 알고리즘으로 우선 구현 요망)
   - 업로드한 영수증에서 금액을 읽어내는 **OCR (글자 인식) 매핑**이 없습니다.
   - 아내분이 직접 개인 카드로 쓴 작업용 지출을 떼어가는 멀티 사용자 레이어도 없습니다.
   - [이중 차단]: 사업비로 태깅하는 순간 가계부 소비 내역에서 스위칭(제거)되는 회계 무결성 로직이 없습니다.

2. **선택형 상황 인지 UI 및 워크-라이프 밸런스 뷰**
   - 장부가 아닌 '삶의 관리'를 위해, 대시보드를 **[성장 모드 (Growth Mode)]**로 수동 전환하여 '자기계발 진도/시급 가치 환산'을 보여주는 라우팅 뷰가 필요합니다. (사용자가 원할 때 켤 수 있는 대시보드 옵션 형태)
   - 저축액에 따라 자동차 사진이 선명해지는 **비전 보드** 화면, 부부 간 집안일 미션 완료 시 용돈을 주는 **홈 앱스 리워드** 로직, 피로도에 따른 알림 톤다운 등 **감성적 UX** 기능이 모두 미구현 상태입니다.

3. **전문 비즈니스 / 인맥 관리 인프라**
   - (제외됨): 라이팅 플롯/픽스쳐 기능
   - 프리랜서 보호용 1페이지 간이 계약 생성 및 사인 **(Legal Lite)** 기능 없음.
   - 세무사 전용 엑셀/증빙 파일 묶음 다운로드 **(Accountant Portal)** 및 세무 내역 기반 지능형 공제 패턴 학습 봇 기능 없음.
   - 클라이언트 파트너사별 평균 입금 지연일수(Lead-Time) 추적 기능 없음.

### 👨‍💻 다음 진행 단계 (마일스톤 수립)

위 정리된 "필수 기능"들을 기반으로, 다음 **4-Step (Research -> Design -> Plan -> Execute)**을 순차적으로 진행합니다. 

무엇부터 설계(Design)를 시작할지 하나 정해주세요.
1. **[세무/안전망 기초]**: 3개월 버퍼 게이지 UI 추가 및 종소세(15~20%) 예비비 박스 분리 로직 설계.
2. **[크루 페이 검증]**: 일당 지급 대장과 은행/엑셀 출금 내역을 대조해 주는 'Bank Sync Verifier(통장 대조기)' 설계.
3. **[스마트 태깅 Lite]**: 시간을 적게 잡아먹으면서도 효과적인 '결제내역 ↔ 캘린더 매칭 추천 알고리즘' 가벼운 버전 설계.
4. **[성장 모드 대시보드]**: 사용자가 스위치로 켜고 끌 수 있는 '성장/미션/비전 보드 뷰' UX 레이아웃 설계.
