# 상세 분석: 반쪽짜리(부분 구현) 기능 현황 및 보완 필요성

시스템을 샅샅이 분석한 결과, 구현은 되어 있으나 V1.1~V2.7 명세서가 요구하는 **'디테일'이나 '깊이'에 도달하지 못한(부분 구현된) 기능들**을 아래와 같이 정리했습니다. 사용자님께서 보시고 **"이건 이대로 써도 충분해(보류/삭제)"** 혹은 **"이건 명세서대로 끝까지 디테일을 살려야 해(진행)"**를 판단해 주시면 됩니다.

---

### 1. 가계부 및 금액 대시보드 (Dual View)
- **현재 구현 상태:** `Home` 화면이 좌(가계) 우(사업) 50:50으로 나뉘어 데이터를 보여줍니다. 지출 한도(Budget) 및 이번 달 사용액도 표시됩니다.
- **명세서와의 차이점 (부분 구현/누락):**
  - **가상 월급제 부재:** 사업 계좌에서 번 돈 중 얼마를 '가상 월급'으로 정해서 가계부 쪽 예산(Budget)으로 꽂아줄지 결정하고 이체 처리하는 로직이 없습니다. 현재 대시보드의 Budget은 UI에 하드코딩된 가짜 데이터(Placeholder)입니다.
  - **3개월 버퍼 (Safety Net) 미적용:** 고정 생활비 3개월치가 안전하게 모였는지 시각적으로 게이지화하는 로직이 없습니다.
- **🤔 체크리스트:**
  - [ ] 가상 월급제 로직을 실데이터 기반으로 끝까지 구현할까요?
  - [ ] 3개월 버퍼 시각화가 대시보드 메인에 꼭 필요할까요?

### 2. 세무(VAT) 및 고정비 분리
- **현재 구현 상태:** 부가세를 분리해서 적립해야 함을 보여주는 `VatReserveCard`와 구독료를 달력에 보여주는 `RecurringExpenseCalendar`가 있습니다.
- **명세서와의 차이점 (부분 구현/누락):**
  - **종소세(15~20%) 파우치 부재:** 부가세 10% 외에 종합소득세 예비비를 추가로 분리해두는 '두 번째 세금 박스' 로직이 없습니다.
  - **사업용 vs 가계용 구독/고정비 분리 미흡:** 구독료(어도비 등) 지출이 발생했을 때, 이것이 가계부의 소비인지 사업용(비용 처리)인지 레이어가 명확히 갈리지 않습니다.
- **🤔 체크리스트:**
  - [ ] 종소세(15~20%) 예비비를 화면에 강제로 분리해서 보여주는 로직이 필요할까요?
  - [ ] 구독료를 사업용/개인용으로 엄격하게 나누는 UI 뎁스를 추가할까요?

### 3. 크루 인건비 지급 검증 (Payroll)
- **현재 구현 상태:** 현장 일당 대장에서 크루 페이를 `지급 완료(Paid)` 상태로 클릭해서 바꿀 수 있습니다.
- **명세서와의 차이점 (부분 구현/누락):**
  - **통장 대조(Bank Sync Verifier) 부재:** 앱에서 '지급 완료'를 눌렀더라도, 실수로 은행에서 송금을 안 했을 수도 있습니다. 실제 은행/액셀 출금 내역과 앱 내 지급 버튼 상태를 교차 검증해 주는 로직이 없습니다.
- **🤔 체크리스트:**
  - [ ] 그냥 앱에서 버튼 누르는 것으로 끝낼까요, 아니면 엑셀/은행 내역과 크루 페이 대장을 교차 검증하는 기능까지 필요할까요?

### 4. 엑셀 장부와 태깅 (Ledger & Categories)
- **현재 구현 상태:** 엑셀이나 카카오뱅크 내역을 복사/붙여넣기해서 웹에 업로드(`ledger-import`)하고, 소비 카테고리(식비, 주유비 등)를 나눌 수 있습니다. 미분류 항목을 걸러내는 화면도 있습니다.
- **명세서와의 차이점 (부분 구현/누락):**
  - **업무 스케줄(캘린더) 연동 모듈 부족:** 명세서의 핵심인 "이 식대가 캘린더 상의 현장 촬영 날에 결제됐으니 사업비로 자동 태깅해 주는" **지능형 알고리즘**이 없습니다. 사용자가 일일이 수동으로 사업/개인을 태깅해야 합니다.
  - **아내분 전용 레이어 미흡:** 남편 카드에 섞인 아내분의 지출을 아내분이 직접 떼어가는 멀티 사용자 UX가 없습니다.
- **🤔 체크리스트:**
  - [ ] (최우선순위) 식대/기름값을 캘린더나 프로젝트 기간과 엮어서 **'현장 소모품인지 개인 지출인지'** 자동 스위칭해 주는 알고리즘 단계를 구현할까요?
