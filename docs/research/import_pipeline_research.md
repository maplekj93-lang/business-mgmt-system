## 2. 주요 에러 분석

### A. BC카드 식별 실패 (`Approve20260217.xls`)
- **현상**: `bc-card.ts` 프로파일이 존재함에도 불구하고 식별되지 않음.
- **원인**: 현재 `bc-card.ts`의 키워드는 `["이용카드", "가맹점명"]`이나, 샘플 파일의 헤더에 개행(`\n`)이나 탭(`\t`)이 포함되어 있어 단순 `includes`로 매칭되지 않음.
- **특이사항**: `A429`, `A010` 등 카드번호 정보를 통해 본인/아내 명의 구분이 가능함.

### B. 현대카드 (제외)
- 사용자 확인 결과 소스가 잘못되었으며 거의 사용하지 않으므로 이번 작업에서 제외함.

### C. 카카오뱅크 런타임 에러
- **현상**: `parseBankSheet` 도중 `TypeError` 발생.
- **원인**: 엑셀 상단부의 빈 셀(null)이 헤더 탐색 대상에 포함되면서 `h.includes(k)` 호출 시 `h`가 null인 경우 크래시 발생.

## 3. 해결 방향

- **키워드 완화**: `hyundai-card.ts`, `bc-card.ts` 등 프로파일의 `keywords`를 실제 샘플 파일의 헤더와 일치하도록 조정.
- **파서 견고성 보강**: `parser.ts`에서 헤더 매칭 시 `h?.includes(k)`와 같이 optional chaining을 사용하거나 정규화를 강화하여 `null/undefined` 에러 방지.
- **신규 프로파일 검증**: `Approve*.xls`가 BC카드임을 명확히 하고, 해당 파일의 특수 구조(개행 포함 헤더 등)를 처리할 수 있도록 `bc-card.ts` 업데이트.
- **raw_description 캡처**: 모든 파서 프로파일에서 `description` 정규화 전의 원본 값을 `raw_description`으로 넘겨주도록 수정.
